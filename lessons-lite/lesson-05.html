05
Плагины и маркетплейс навыков
Архитектура

Агент из прошлых уроков умеет работать с инструментами. Но все инструменты зашиты в него намертво. Хотите добавить новый — переписывайте агента. Skills (навыки) решают эту проблему: агент подключает возможности как приложения на смартфоне.

Аналогия: App Store для ИИ

Представьте, что ваш агент — это смартфон. Из коробки он умеет звонить и писать SMS. Полезно, но ограниченно. А теперь установите приложения — навигатор, банк, переводчик — и он становится в 100 раз полезнее. При этом вам не нужно покупать новый телефон каждый раз.

Skills = приложения для агента. Каждый skill — это набор инструментов и инструкция, упакованные как плагин. Агент не загружает всё сразу — сначала видит список доступных навыков, потом активирует только те, которые нужны для конкретной задачи.

В основе модульности лежит принцип progressive disclosure (постепенное раскрытие). Агент не читает инструкции ко всем 50 навыкам сразу — это было бы как открыть 50 учебников одновременно. Вместо этого он действует в три шага:

Discovery → Activation → Execution

Discovery (обнаружение): агент видит краткий каталог — название и описание каждого навыка в одну строку. Это как список приложений в App Store: иконка, название, краткое описание.

Activation (активация): агент выбирает нужный навык и загружает полную инструкцию. Это как открыть приложение — только сейчас агент получает детальные правила работы.

Execution (выполнение): агент использует инструменты навыка для решения задачи. Как вы используете навигатор — вводите адрес, получаете маршрут.

Зачем такая сложность? Представьте, что у агента 50 навыков. Загрузить все инструкции сразу — это как заставить сотрудника прочитать 50 инструкций перед началом работы. С progressive disclosure агент сначала пробегает глазами каталог, активирует 2-3 нужных навыка и сразу приступает к делу. Быстрее, дешевле, точнее.

Пример: анализ Excel-файла
Пользователь Проанализируй файл отчёт.xlsx
Система Discovery: доступны навыки — filesystem, analytics, web-search, email
Агент Мне нужен файл и аналитика. Активирую навыки: filesystem, analytics
filesystem Файл отчёт.xlsx прочитан: 1200 строк, 8 столбцов, данные о продажах
analytics Топ-3 категории: электроника (42%), одежда (28%), продукты (18%). Рост +15% к прошлому кварталу
Агент Готово. Вот сводка: выручка выросла на 15%, лидер — электроника. Подробный отчёт прилагаю.

Обратите внимание: агент не загружал навык web-search и email — они были не нужны для этой задачи. Это и есть экономия progressive disclosure. Для другого запроса набор активных навыков будет совсем другим.

Бизнес-пример: маркетплейс навыков в компании

Представьте крупную компанию. IT-отдел создаёт skill для работы с 1С, HR-отдел — для кадрового учёта, маркетинг — для аналитики рекламных кампаний. Все навыки публикуются на внутреннем маркетплейсе.

Теперь любой сотрудник может подключить агента к нужным навыкам: бухгалтер активирует 1С + отчётность, маркетолог — аналитику + CRM, руководитель — дашборды + почту. Один агент, разные конфигурации.

Ещё важнее — навыками можно делиться между компаниями. Если партнёр создал skill для работы с Bitrix24 — вы подключаете его к своему агенту за минуту. Не нужно разрабатывать интеграцию с нуля.

Критерий Монолитный агент Агент с плагинами
Добавить новый инструмент Переписать код агента Подключить готовый skill
Количество возможностей Ограничено при создании Растёт с маркетплейсом
Потребление ресурсов Все инструменты загружены всегда Только нужные активированы
Переиспользование Инструменты зашиты в одного агента Навыки работают в любом агенте
Обновление Обновить весь агент целиком Обновить один skill независимо
Главное из этого урока
  • Skills делают агента расширяемым. Новые возможности подключаются как приложения, без переписывания самого агента.
  • Progressive disclosure экономит ресурсы. Агент загружает только те навыки, которые нужны для конкретной задачи. Не все 50 сразу.
  • Можно создавать свои навыки. Компания может упаковать свою экспертизу (работа с 1С, CRM, внутренние процессы) в skills и распространять их между командами и партнёрами.
  • Маркетплейс навыков — это экосистема. Чем больше skills создано, тем полезнее становится каждый агент.
Ключевая мысль: Сила агента — не в том, что он умеет из коробки, а в том, какие навыки он может подключить. Точно так же, как ценность смартфона определяется не заводскими приложениями, а магазином приложений с миллионами вариантов.
